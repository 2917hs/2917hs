GlobalLogic — HHAExchange

HHAExchange is a leading home-health management platform in the United States, serving close to a million users — caregivers, coordinators, and administrators — across scheduling, Electronic Visit Verification, and mobile care coordination. It's a very large-scale, compliance-heavy product, and I was brought in as part of the GlobalLogic team to work on several major streams simultaneously.

My first focus was the web portal. There were legacy reporting modules that were built on old ASPX technology, and the team was spending a lot of time manually collecting and preparing data. I re-architected those modules using Angular and TypeScript on the frontend, backed by .NET 8 Web APIs and MS SQL Server. Beyond just modernising the stack, I did serious SQL optimisation work — indexed views, execution plan tuning, keyset pagination — which cut query latency significantly and made the reports feel almost instant compared to before.

At the same time, I was involved in one of the most complex pieces of work I've done: planning and executing a zero-downtime migration across three production servers, which together served around 950,000 users. We couldn't afford any downtime, so we designed a risk-layered strategy — starting with the smallest server, validating with automated health-gate checkpoints, and only moving to the next when everything looked stable. We used feature flags and backward-compatible schema migrations to ensure the old and new versions could coexist on the same database during the transition.

Then I moved on to building a completely greenfield mobile backend in Python on AWS Lambda. This was an event-driven architecture using Lambda-to-Lambda communication and Amazon SNS and SQS as our messaging backbone — we replaced Kafka with this approach because it integrated natively with the rest of our AWS infrastructure and reduced operational overhead considerably. I also brought in AWS Secrets Manager to centralise credential management, which eliminated hardcoded secrets across the codebase entirely.

For the mobile frontend, I used Ionic with Angular — it meant we could reuse a large portion of the Angular services and RxJS patterns already built for the web portal, saving around 40% of development effort. The mobile app had full feature parity: scheduling, real-time messaging, and file uploads via S3 pre-signed URLs — all going through an AWS AppSync GraphQL BFF layer. I also worked closely with the DevOps team to set up the entire CI/CD infrastructure using GitHub Actions, with Docker multi-stage builds and Terraform for infrastructure provisioning, and mentored four engineers on the team.

IBM — Baxter International (ExactaMix)

Baxter International is a global medical device and healthcare company, and the product I worked on was ExactaMix — an automated compounding system used in hospitals across the US and Europe for clinical nutrition. This is a highly regulated, safety-critical environment, so every change goes through strict compliance and regulatory review.

The core of the application was a Windows Forms application built in VB.NET and C#. One of the more significant compliance-driven changes I made was replacing the legacy barcode scanning mechanism in the order processing workflow with Data Matrix reading — a regulatory requirement that had to be implemented precisely and validated thoroughly.

The bigger strategic work was the cloud migration. We adopted Domain-Driven Design as the guiding principle, and I picked up modules one by one — Stocks, Inventory, Orders, Customer management, and Admin — rebuilding each as an independent microservice in C# and .NET 6, deployed as AWS Lambda functions. We used SQS for all the asynchronous work, with Lambda writing records to a SQL database and Hangfire background services picking those up and publishing them onward. Once a module was stable, we removed it from the desktop application entirely. Other internal teams also started consuming those services.

There was also an interesting integration challenge: a separate application needed to consume our order data. I solved that using Power Automate Desktop, consuming the order payload in JSON and orchestrating creation entirely in the background. I also built a localisation pipeline using .NET resource files and Roslyn compiler APIs, backed by an Angular web app for non-technical stakeholders to manage translations themselves. Finally, I set up Azure DevOps CI/CD pipelines and wrote PowerShell scripts to handle library group migrations.

Wolters Kluwer — Lippincott Journals

Lippincott Journals is part of Wolters Kluwer, one of the world's leading academic and medical publishing companies. The platform I worked on is used by researchers, authors, doctors, medical staff, and university students to access peer-reviewed scientific and medical literature. It's a high-traffic platform where reliability and user experience really matter.

My day-to-day work spanned the full frontend stack — Angular, TypeScript, JavaScript ES6, HTML, and CSS — delivering new features and fixing bugs. The codebase was mature and large, so navigating it carefully without introducing regressions was an important part of the job.

One of the more technically interesting pieces was the ETOC email notification system. These are long-running batch jobs, and without careful management they can easily overwhelm system resources. I implemented this using .NET Hosted Services with backpressure techniques to throttle the processing rate, ensuring the jobs ran reliably without degrading the rest of the platform.

I also built an article sharing and print-to-PDF module rolled out across the entire platform, and delivered an MVP for a new payment strategy using CyberSource Flex Microform — a PCI-compliant inline card capture solution that keeps the user on the platform rather than redirecting externally. Finally, I set up the CI/CD pipelines for Azure Functions workloads using Azure DevOps.

Accenture — Metropolitan Police (CRIS/Connect)

CRIS — the Crime Recording Information System — is the Metropolitan Police Service's core crime recording platform, one of the most sensitive and operationally critical systems in UK law enforcement. Working on this was a very different experience to commercial software — the data is extremely sensitive, the processes are tightly governed, and the stakes of getting something wrong are very high.

My work involved delivering enhancements and fixing bugs across the C#, JavaScript, jQuery, HTML, and CSS codebase, along with SQL query optimisation. I was part of the broader effort to migrate the legacy on-premises desktop application to a web application hosted on Azure.

The piece I'm most proud of is the GDPR-compliant data masking application I built. The team needed access to realistic criminal records data for AI integration and automation work, but directly accessing live operational data raised serious legal and ethical concerns. I built a tool that applied masking rules to anonymise raw data so it could be safely used for testing and AI model training without exposing real individuals' information.

The other significant piece was a background service to synchronise user access between Entra ID and on-premises Active Directory across 40,000 users. In a hybrid environment like the Met's, keeping identity and permissions consistent across both systems is critical for security and operational continuity. The service ran automatically, resolving discrepancies without any manual intervention.
